# portal-index-notes

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放各类独立的 HTML 页面资源。这些页面可能包含信息索引、笔记整理、工具页面或静态内容展示等。每个页面均以独立 HTML 文件形式存在，不依赖特定后端服务或数据库。

## 目录说明

```
portal-index-notes/
├── index.html          # 主入口页面（可选）
├── pages/              # 存放独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源文件（CSS/JS/图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：核心目录，每个 HTML 文件代表一个独立页面。
- **assets/**：存放页面共用的样式、脚本或静态资源。
- **index.html**：可选的导航或入口页面，用于快速访问各子页面。

## 页面归档说明

- 每个页面均以 `.html` 文件独立保存，文件名建议使用有意义的英文或数字组合（如 `notes-2024.html`）。
- 页面内容可包含文本、图片、链接、表格等常见 HTML 元素，不限定具体格式。
- 所有页面均为静态内容，可直接通过浏览器打开或部署到任何静态托管服务。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 `.html` 文件，并确认资源引用路径正确。
- 更新页面：直接编辑对应 HTML 文件，保持文件结构清晰。
- 删除页面：移除对应文件，并检查 `index.html` 或其他页面中的链接是否需要更新。
- 建议保持文件命名规范和目录结构一致，便于长期维护。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/portal-index-notes.git
   ```
2. 在 `pages/` 目录下添加或编辑 HTML 文件。
3. 直接在浏览器打开 `index.html` 或 `pages/` 下的任意文件即可预览。

本项目仅用于内容归档与展示，不涉及特定许可协议。引用外部资源时请遵守其原始许可条款。
